# JobPilot AI Design Specification

更新时间：2026-06-24

## 1. 设计定位

JobPilot AI 是一个岗位作战与面试知识库系统。长期面向各行各业求职者，第一版 MVP 聚焦计算机领域求职者，把简历、岗位 JD、岗位作战卡、题库、练习、知识库和投递复盘串成完整闭环。

产品体验不做营销型首页，登录后的第一屏就是工作台。界面应像一个安静、可靠、可反复使用的求职作战台，而不是一次性 AI 生成器。

## 2. 体验目标

- 让用户知道当前最应该做什么。
- 让岗位、作战卡、题目、练习和知识库之间的关系清楚可见。
- 让 AI 能力以“可验证、可保存、可复盘”的方式出现。
- 让第一版服务计算机领域，同时保留多行业扩展能力。
- 让页面足够紧凑，适合反复查看、筛选、练习和复盘。

## 3. 设计原则

### 3.1 岗位为中心

用户不是来管理孤立的简历和题目，而是围绕一个目标岗位准备。所有核心页面都要能回到岗位上下文：

- 当前岗位
- 使用的简历版本
- 对应的岗位作战卡
- 推荐题目
- 练习记录
- 知识库沉淀
- 投递状态

### 3.2 AI 结果必须结构化

AI 输出不能只是一大段文本。作战卡、题目生成、答案评分和知识条目整理都要结构化展示，方便用户判断、编辑、保存和复习。

### 3.3 题库优先的体验要可见

生成题目时，页面要让用户理解系统不是直接乱生成，而是先检索公共题库，再用 AI 补充缺口。

推荐展示：

- 公共题库召回数量
- AI 补充数量
- 候选题入库状态
- 标签覆盖情况

### 3.4 个人资产沉淀优先

用户每次练习后的高质量结果都应该能沉淀到个人知识库。知识库不是附属页面，而是用户长期复习和面试前回看的核心资产。

### 3.5 当前计算机领域，长期多行业

第一版页面可以默认展示计算机方向标签，例如 Java、Spring Boot、MySQL、Redis、Docker、React、AI 应用等。但信息架构、字段命名和筛选器要使用更通用的语言：

- 行业领域
- 岗位方向
- 能力标签
- 题目类型
- 经历类型

## 4. 信息架构

```text
JobPilot AI
├─ Dashboard
├─ 简历
│  ├─ 简历列表
│  ├─ 上传简历
│  ├─ 文本简历
│  └─ 简历详情
├─ 岗位
│  ├─ 岗位列表
│  ├─ 新建岗位
│  └─ 岗位详情
├─ 岗位作战卡
│  ├─ 匹配分析
│  ├─ 能力拆解
│  ├─ 补强计划
│  └─ 推荐生成题目
├─ 题库与练习
│  ├─ 专属题目
│  ├─ 公共题库
│  ├─ 题目详情
│  └─ 练习评分
├─ 个人知识库
│  ├─ 知识条目
│  ├─ 错题总结
│  ├─ 项目话术
│  └─ 面试复盘
├─ 投递记录
└─ 设置
```

## 5. 导航结构

采用后台工作台式布局：

- 左侧固定导航
- 顶部上下文栏
- 主内容区
- 可选右侧上下文面板

左侧导航：

```text
Dashboard
简历
岗位
题库练习
知识库
投递复盘
设置
```

顶部上下文栏：

- 当前用户
- 当前默认简历
- 当前活跃岗位
- 快捷操作：新建岗位、上传简历、生成作战卡

## 6. 关键页面

第一版必须设计以下页面：

- `/login`
- `/register`
- `/dashboard`
- `/resumes`
- `/jobs`
- `/jobs/:id`
- `/battle-cards/:id`
- `/questions`
- `/questions/:id`
- `/knowledge`
- `/applications`
- `/settings`

详细页面规格见 [页面设计规格](docs/design/page-specs.md)。

## 7. 核心用户流

核心闭环：

```text
注册登录
-> 录入简历
-> 录入岗位
-> 生成作战卡
-> 生成题目
-> 练习评分
-> 保存知识
-> 记录投递
-> 复盘改进
```

详细用户流见 [用户流程设计](docs/design/user-flows.md)。

## 8. 视觉方向

整体风格：

- 专业
- 克制
- 清晰
- 信息密度适中
- 强调状态和行动

避免：

- 大面积营销型 hero
- 装饰性渐变背景
- 过多卡片堆叠
- 单一蓝紫色主题
- 无意义插画

推荐视觉语言：

- 中性色作为主背景
- 蓝色用于主要行动和信息
- 绿色用于优势、通过、掌握
- 黄色或琥珀色用于风险和待复习
- 红色用于错误、短板和失败原因

详细组件与视觉规范见 [组件系统](docs/design/component-system.md)。

## 9. AI 交互策略

AI 出现的地方：

- 岗位作战卡生成
- 专属题目补充生成
- 答案评分
- 优化回答
- 面试官追问
- 知识条目整理
- 投递复盘建议

AI 交互必须包含：

- 明确的生成中状态
- 可重试
- 可查看原始依据
- 可编辑保存
- 失败兜底文案
- 成本和限流提示

详细状态规范见 [交互状态设计](docs/design/interaction-states.md)。
