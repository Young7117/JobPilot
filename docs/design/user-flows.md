# JobPilot AI 用户流程设计

更新时间：2026-06-24

## 1. 主流程

```mermaid
flowchart TD
    A["注册或登录"] --> B["录入简历"]
    B --> C["录入岗位 JD"]
    C --> D["生成岗位作战卡"]
    D --> E["系统提取行业、岗位方向、能力标签和短板"]
    E --> F["公共题库优先检索"]
    F --> G{"题目数量和覆盖度足够？"}
    G -->|是| H["生成专属题目清单"]
    G -->|否| I["AI 补充生成缺口题目"]
    I --> J["进入候选题库"]
    J --> H
    H --> K["用户练习并提交答案"]
    K --> L["AI 评分、优化、追问"]
    L --> M["保存优质回答到个人知识库"]
    M --> N["记录投递结果"]
    N --> O["AI 生成复盘建议"]
```

## 2. 首次使用流程

目标：让新用户尽快从空状态进入第一张岗位作战卡。

```mermaid
flowchart TD
    A["进入 Dashboard 空状态"] --> B["引导上传或粘贴简历"]
    B --> C["解析简历并保存"]
    C --> D["引导创建第一个岗位"]
    D --> E["选择默认简历"]
    E --> F["生成岗位作战卡"]
    F --> G["展示匹配分和下一步行动"]
```

关键设计点：

- Dashboard 空状态只保留一个主行动：开始准备一个岗位。
- 简历录入支持文件上传和文本粘贴，文件解析失败时直接进入文本编辑模式。
- 生成作战卡前明确展示输入来源：简历版本和岗位 JD。

## 3. 简历录入流程

```mermaid
flowchart TD
    A["点击新增简历"] --> B{"选择录入方式"}
    B -->|上传 PDF| C["解析文本型 PDF"]
    B -->|上传 DOCX| D["解析 DOCX"]
    B -->|粘贴文本| E["进入文本编辑器"]
    C --> F{"解析成功？"}
    D --> F
    F -->|是| G["展示解析结果"]
    F -->|否| E
    E --> G
    G --> H["用户编辑内容"]
    H --> I["保存简历版本"]
    I --> J["可设为默认简历"]
```

关键状态：

- 解析中
- 解析失败
- 内容为空
- 版本保存成功
- 默认简历已变更

## 4. 岗位作战卡生成流程

```mermaid
flowchart TD
    A["岗位详情页"] --> B["点击生成作战卡"]
    B --> C["选择简历版本"]
    C --> D["确认行业领域和岗位方向"]
    D --> E["生成请求 hash"]
    E --> F{"Redis 是否命中缓存？"}
    F -->|是| G["读取缓存结果"]
    F -->|否| H["调用大模型生成"]
    H --> I{"JSON 是否可解析？"}
    I -->|是| J["保存作战卡"]
    I -->|否| K["提示失败并允许重试"]
    G --> J
    J --> L["进入作战卡详情页"]
```

页面必须展示：

- 匹配分
- 核心要求
- 技能或能力拆解
- 已有优势
- 明显短板
- 简历优化建议
- 面试重点
- 三天补强计划
- 七天补强计划
- 投递风险提醒

## 5. 题目生成流程

```mermaid
flowchart TD
    A["作战卡详情页"] --> B["点击生成专属题目"]
    B --> C["提取能力标签和短板"]
    C --> D["Qdrant 检索公共题库"]
    D --> E["按语义、标签、质量分排序"]
    E --> F{"覆盖度足够？"}
    F -->|是| G["生成题目清单"]
    F -->|否| H["识别缺口能力点"]
    H --> I["AI 补题"]
    I --> J["相似题去重"]
    J --> K["质量评分"]
    K --> L["敏感信息过滤"]
    L --> M["进入候选题库"]
    M --> G
```

题目清单应标注来源：

- 公共题库
- AI 补充
- 候选题

## 6. 练习评分流程

```mermaid
flowchart TD
    A["题目详情页"] --> B["用户输入答案"]
    B --> C["提交评分"]
    C --> D["AI 对比参考答案和岗位重点"]
    D --> E["返回分数、优点、问题、优化答案、追问"]
    E --> F{"用户是否保存？"}
    F -->|保存| G["整理为知识库条目"]
    F -->|不保存| H["保留练习记录"]
    G --> I["进入个人知识库"]
```

关键设计点：

- 用户答案编辑区和 AI 反馈区左右或上下并列。
- AI 优化答案必须可复制、可编辑、可保存。
- 掌握状态需要就近操作：未练习、需复习、已掌握、高频重点。

## 7. 投递复盘流程

```mermaid
flowchart TD
    A["创建投递记录"] --> B["关联岗位"]
    B --> C["关联简历版本和作战卡"]
    C --> D["记录投递状态"]
    D --> E{"是否有结果？"}
    E -->|无| F["等待更新"]
    E -->|有| G["记录面试结果和失败原因"]
    G --> H["AI 生成复盘建议"]
    H --> I["保存到知识库或复习计划"]
```

投递状态建议：

```text
准备中
已投递
笔试中
面试中
已通过
已拒绝
已放弃
```
