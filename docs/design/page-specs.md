# JobPilot AI 页面设计规格

更新时间：2026-06-24

## 1. 全局布局

登录后采用工作台布局：

```text
┌──────────────┬────────────────────────────────────┐
│ 左侧导航      │ 顶部上下文栏                         │
│              ├────────────────────────────────────┤
│              │ 主内容区                             │
│              │                                    │
└──────────────┴────────────────────────────────────┘
```

桌面端：

- 左侧导航宽度：224px
- 顶部栏高度：56px
- 内容区最大宽度：1440px
- 主内容区内边距：24px

移动端：

- 左侧导航收起为底部或抽屉导航
- 表格转为列表卡片
- 双栏布局改为上下堆叠

## 2. `/login`

页面目标：让用户快速登录进入工作台。

布局：

- 左侧品牌区域：产品名、简短定位
- 右侧登录表单

表单字段：

- 用户名或邮箱
- 密码

主操作：

- 登录
- 去注册

状态：

- 登录中
- 用户名或密码错误
- 登录过期后重新登录提示

## 3. `/register`

页面目标：完成账号创建。

表单字段：

- 用户名
- 邮箱
- 密码
- 确认密码

主操作：

- 注册并进入 Dashboard

设计要求：

- 密码规则就近提示
- 注册成功后自动登录

## 4. `/dashboard`

页面目标：展示当前备战进度，并引导下一步行动。

首屏结构：

- 顶部欢迎区：当前默认简历、活跃岗位、下一步建议
- 数据概览：简历数量、岗位数量、作战卡数量、题目数量、待复习题目、投递状态
- 进行中的岗位列表
- 本周待复习题目
- 最近知识库条目

空状态：

- 没有简历：主按钮“录入第一份简历”
- 有简历但没有岗位：主按钮“创建目标岗位”
- 有岗位但没有作战卡：主按钮“生成岗位作战卡”

组件：

- Statistic
- Table 或 List
- Progress
- Tag
- Button

## 5. `/resumes`

页面目标：管理多个简历版本。

页面结构：

- 顶部操作区：上传 PDF、上传 DOCX、粘贴文本
- 简历列表
- 简历详情抽屉或详情页

列表字段：

- 标题
- 目标岗位
- 文件类型
- 版本号
- 是否默认
- 更新时间
- 操作

详情区：

- 解析文本
- 可编辑内容
- 设置默认
- 删除

关键状态：

- 文件上传中
- 解析中
- 解析失败
- 不支持扫描版 PDF
- 内容为空

## 6. `/jobs`

页面目标：管理目标岗位。

页面结构：

- 顶部筛选：行业领域、岗位方向、状态、关键词
- 新建岗位按钮
- 岗位列表

列表字段：

- 公司
- 岗位名称
- 城市
- 薪资
- 行业领域
- 岗位方向
- 状态
- 是否已有作战卡
- 更新时间

岗位状态：

```text
准备中
已投递
面试中
已结束
已归档
```

## 7. `/jobs/:id`

页面目标：展示岗位详情，并作为生成作战卡的入口。

页面结构：

- 岗位基础信息
- JD 内容
- 关联简历选择
- 当前作战卡状态
- 推荐操作

主操作：

- 生成岗位作战卡
- 查看作战卡
- 生成专属题目
- 创建投递记录

设计要求：

- 如果没有默认简历，先提示选择简历。
- 如果已有作战卡，展示最近生成时间和匹配分。

## 8. `/battle-cards/:id`

页面目标：让用户清楚理解自己和岗位的匹配关系，并进入补强和练习。

页面结构：

- 顶部摘要：公司、岗位、匹配分、生成时间
- 左侧主内容：核心要求、能力拆解、优势、短板、面试重点
- 右侧行动栏：三天计划、七天计划、风险提醒、生成题目按钮

模块：

- 匹配分仪表或进度环
- 核心要求列表
- 能力标签矩阵
- 优势和短板对照
- 简历优化建议
- 面试重点
- 补强计划
- 风险提醒

主操作：

- 生成专属题目
- 保存建议到知识库
- 返回岗位详情

## 9. `/questions`

页面目标：查看和筛选题目，进入练习。

页面结构：

- 筛选区：行业、岗位方向、标签、题目类型、难度、掌握状态、关联岗位
- 题目列表
- 批量操作

题目来源标记：

- 公共题库
- AI 补充
- 候选题

列表字段：

- 标题
- 题型
- 难度
- 标签
- 来源
- 掌握状态
- 最近练习时间

## 10. `/questions/:id`

页面目标：完成单题练习、AI 评分和知识沉淀。

页面结构：

- 题目内容
- 标签、难度、来源
- 参考答案折叠区
- 用户答案编辑器
- AI 评分结果
- AI 优化回答
- 面试官追问
- 练习历史

主操作：

- 提交评分
- 保存到知识库
- 标记掌握状态
- 下一题

设计要求：

- 参考答案默认折叠，避免用户直接照抄。
- AI 评分结果必须分块展示，不能只是一段长文本。
- 优化回答支持编辑后保存。

## 11. `/knowledge`

页面目标：沉淀和检索个人面试资产。

页面结构：

- 左侧分类
- 顶部搜索和标签筛选
- 知识条目列表
- 详情或编辑抽屉

分类：

- 项目话术
- 技术笔记
- 错题总结
- 面试复盘
- HR 回答
- 简历素材

列表字段：

- 标题
- 分类
- 标签
- 掌握程度
- 来源
- 更新时间

主操作：

- 新建知识条目
- 编辑
- 标记掌握程度
- 关联岗位或题目

## 12. `/applications`

页面目标：记录投递进度，并将结果转化为复盘建议。

页面结构：

- 状态统计
- 投递记录列表
- 新建投递记录
- 结果复盘区

列表字段：

- 公司
- 岗位
- 状态
- 投递日期
- 面试日期
- 结果
- 失败原因
- 关联作战卡

主操作：

- 新建投递记录
- 更新状态
- 生成复盘建议
- 保存复盘到知识库

## 13. `/settings`

页面目标：管理账号、AI 配置和偏好。

第一版设置项：

- 账号信息
- 默认简历
- 默认行业领域
- 默认岗位方向
- AI 生成偏好
- 退出登录
